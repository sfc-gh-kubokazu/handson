# STEP 2: データロードの準備

## このセクションでやること

データをロードする前に必要な「入れ物」と「設定」を作成します。

- **実行ファイル**: `../scripts/01_data_load_prep.sql`

---

## Snowflakeの概念: データ格納の階層構造

```
アカウント
└── データベース (例: citibike)
    └── スキーマ (例: public)
        ├── テーブル (例: trips)
        ├── ビュー
        ├── ステージ       ← 外部データへの「参照」
        └── ファイルフォーマット ← データ形式の定義
```

### データベース
- テーブルやビューを論理的にまとめるコンテナ
- 用途やプロジェクトごとに分けるのが一般的
- 例: `citibike`（乗車データ）/ `weather`（気象データ）

### スキーマ
- データベース内のさらに細かい分類
- デフォルトは `public`
- 環境分離（`dev` / `prod`）やチーム分割に使う

### ステージ（Stage）
Snowflake でデータをロードする際、直接アップロードするのではなく「ステージ」を経由します。

| ステージ種別 | 説明 |
|---|---|
| **内部ステージ** | Snowflake が管理するストレージ（`PUT` コマンドでアップロード） |
| **外部ステージ** | S3 / Azure Blob / GCS への参照（認証情報を Snowflake に登録） |
| **テーブルステージ** | テーブルに紐づく専用ステージ |

今回は S3 上に公開されているデータを使うので **外部ステージ**を使います。

### ファイルフォーマット
ロードするファイルのフォーマット定義（CSV の区切り文字・エスケープ・ヘッダーの有無など）を  
再利用できるオブジェクトとして保存しておく仕組みです。

---

## 手順

1. `../scripts/01_data_load_prep.sql` をワークシートに貼り付ける（またはファイルから開く）
2. 上から順に **Ctrl+Enter（1行実行）** または **全選択して Run** で実行

### 実行後の確認ポイント

**データベースが作成された:**
```sql
SHOW DATABASES;  -- citibike が表示される
```

**テーブルが作成された:**
```sql
SHOW TABLES IN DATABASE citibike;  -- trips が表示される
```

**ステージに S3 のファイルが見える:**
```sql
LIST @citibike_trips;
-- 多数の .csv.gz ファイルが表示される
```

**ファイルフォーマットが作成された:**
```sql
SHOW FILE FORMATS IN DATABASE citibike;  -- csv が表示される
```

## よくある質問

**Q: ステージのファイル一覧が表示されるのに時間がかかる**  
A: S3 上に大量のファイルがあるため、数十秒かかることがあります。正常です。

**Q: `USE DATABASE citibike;` が失敗する**  
A: ロールが `sysadmin` になっていることを確認してください。  
右上のユーザーアイコン → Switch Role → SYSADMIN
