# STEP 1: Snowflake UI の操作体験

## このセクションでやること

Snowflake の Web UI（Snowsight）の主要な画面を一通り触れてみます。  
ここでは SQL を書かず、まず「何がどこにあるか」を把握することが目的です。

---

## Snowflakeのインターフェイス概要

Snowsight（Snowflake の Web UI）には主に以下のメニューがあります。

### ワークシート
- SQL を書いて実行する場所
- 複数のワークシートをタブで管理できる
- 結果をグラフ化することも可能
- **今日のハンズオンはほとんどここで作業します**

### ダッシュボード
- ワークシートのクエリ結果をビジュアル化してダッシュボードにまとめる機能
- 複数チャートを1画面に配置できる

### データベース
- アカウント内のデータベース・スキーマ・テーブルを一覧で見られる
- テーブルのプレビューやカラム定義も確認できる

### プライベート共有データ
- 他のアカウントから共有されたデータを確認する場所
- Snowflake Marketplace で入手したデータセットもここに表示される

### Snowflake Marketplace
- 外部プロバイダーが公開するデータを取得できるマーケットプレイス
- 天気・地理・金融・医療など様々なデータが揃っている
- データはコピーせずに直接クエリできる（Data Sharing の仕組み）

### クエリ履歴
- 実行したすべての SQL の履歴が確認できる
- 実行時間・スキャンデータ量・キャッシュ利用の有無も見られる
- **デバッグやパフォーマンス分析に活用する**

### ウェアハウス
- コンピュートリソース（= SQL を実行する「エンジン」）を管理
- XS / S / M / L / XL などのサイズがあり、大きいほど高速・高コスト
- **「サスペンド中」は課金されない**

### リソースモニター
- クレジット消費量に上限を設定し、超えたらアラートや停止を自動実行
- コスト管理に使う

### ロール
- Snowflake のアクセス制御の基本単位
- デフォルトで `ACCOUNTADMIN` / `SYSADMIN` / `PUBLIC` などが用意されている
- 右上の「スイッチロール」から切り替えられる

### ユーザー
- アカウント内のユーザー一覧（ACCOUNTADMIN のみ閲覧可）

## ラボストーリー

このハンズオンでは **New York City の Citi Bike** データを使います。

- Citi Bike はニューヨーク市のシェアサイクルサービス
- トリップ（乗車記録）データが S3 に公開されている
- さらに **天気データ（JSON）** と組み合わせて「天気とトリップ数の相関」を分析します

```
[S3: Citibike CSV] → ロード → [Snowflake: citibike.public.trips]
[S3: 気象 JSON]   → ロード → [Snowflake: weather.public.json_weather_data]
                                         ↓
                               クロスデータベース JOIN で相関分析
```
